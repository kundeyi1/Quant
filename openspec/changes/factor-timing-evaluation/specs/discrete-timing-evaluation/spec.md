## ADDED Requirements

### Requirement: TimingTester Instantiation
The system SHALL provide a `core.TimingTester.TimingTester` class constructor that accepts `signal_series` (pd.DataFrame with 0/1 signals) and `benchmark_series` (pd.Series containing underlying asset close prices).

#### Scenario: Correct initialization of TimingTester
- **WHEN** a user instantiates `TimingTester(signal_matrix, benchmark_series, output_dir="results/timing")`
- **THEN** the tester object is successfully created and defaults to evaluating 0/1 specific behavior.

### Requirement: Net Asset Value and Drawdown Visualization
The `TimingTester` MUST plot the strategy Net Asset Value (NAV) curve in red and the normalized Benchmark (e.g., 万得全A) in dark blue on the left axis. It MUST also fill the drawdowns as a grey shadow area mapped to the right axis (max 0%, down to -X%).

#### Scenario: Visualizing timing performance
- **WHEN** the user calls `tester.plot_timing_nav_and_drawdown("布林带_择时净值")`
- **THEN** a matplotlib dual-axis plot is generated showing NAV, normalized Benchmark NAV, and drawdown shadow area, and saved to `results/timing`.

### Requirement: Directory Output Redirection
The system MUST save output analyses and plots generated by `TimingTester` specifically to the user-specified or default directory (`results/timing`).

#### Scenario: Verify isolated output folder
- **WHEN** a timing analysis report or plot is rendered
- **THEN** the system ensures the `results/timing/` directory exists and saves the `.png` or `.csv` there.
